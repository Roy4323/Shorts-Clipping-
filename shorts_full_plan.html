01 — System Overview

Design Philosophy

CORE PRINCIPLE Never use vision models for highlight detection. Treat video as audio + text. Vision models are only invoked for reframing subject detection — using free local models (MediaPipe, OpenCV). Total API cost stays under $0.01 per video.

The pipeline ingests a YouTube URL, downloads at optimal quality, extracts a transcript, runs multi-signal AI scoring to find the best highlight windows, reframes from 16:9 to 9:16 with subject-aware cropping, and burns subtitles — outputting ready-to-post short-form clips.

High-Level Pipeline Flow

Stage 1 — URL Intake & Metadata
No download yet. yt-dlp metadata-only mode. Claude Haiku classifies content type. <2 seconds.
Stage 2 — Parallel Download + Transcript
yt-dlp (720p) and Supadata run concurrently. Transcript returns in seconds. Download is the long pole.
Stage 3 — Multi-Signal Highlight Scoring
Semantic (Claude Haiku) + audio energy (librosa) + speaker turns (pyannote). Combined score per 30s window.
Stage 4 — Clip Windowing & Cutting
Snap to sentence boundaries. Enforce 20–90s duration. FFmpeg stream copy — no re-encode.
Stage 5 — Reframing 16:9 → 9:16
MediaPipe face tracking (podcast) / optical flow (sports) / smart center crop (fallback). 30-frame smoothing.
Stage 6 — Subtitle Burn + Output
SRT from transcript, offset to clip time, burned via ffmpeg. Bold white with black outline. Done.

YouTube URLs: Supadata pulls existing captions, returns timestamped segments in seconds — no transcription compute cost. Fallback for uploaded video (v2): extract audio → faster-whisper locally.

Snap window edges to sentence boundaries. Enforce 20–90s duration. Stream copy — no re-encode.

03 — Highlight Scoring

Multi-Signal Scoring Engine

Three independent signals computed per 30-second window, combined into a composite score.

01
Semantic Scoring
Claude Haiku analyzes the full timestamped transcript. Returns top 5 engaging 30–60s windows with reasoning as JSON.
~$0.003 / 1hr video · Weight: 40%
02
Audio Energy
librosa RMS energy per 0.5s window. Detects laughter, applause, raised voice, emphasis. Runs locally.
Free · Local · Weight: 35%
03
Speaker Turn Density
pyannote.audio counts speaker switches per 30s window. High switch density = conversational energy = engaging.
Free · Local · Weight: 25%

Score Formula

score = (0.40 × semantic) + (0.35 × energy) + (0.25 × dialogue_density)
Content TypeSemanticEnergyOther
Podcast / Interview0.400.350.25 dialogue
Music Instrumental0.000.700.30 onset
Sports / Action0.200.300.50 motion
BONUS SIGNAL sentence-transformers: embed all transcript segments, find those semantically closest to the video's core topic. Most on-topic + high energy = best clip. Prevents off-topic tangents from being selected. Add if time allows.

04 — Reframing

16:9 → 9:16 Subject-Aware Crop

CRITICAL SMOOTHING RULE Raw per-frame crop coordinates produce unwatchable jitter. Always smooth with a rolling average of 24–30 frames before applying the crop. This single step is the difference between professional and amateur output.

Podcast / Interview — MediaPipe Face Tracking

import mediapipe as mp
mp_face = mp.solutions.face_detection.FaceDetection(min_detection_confidence=0.5)

# Per frame:
# 1. Run FaceDetection → get bbox center (x, y)
# 2. If no face: use frame center as fallback
# 3. Smooth: rolling average over deque(maxlen=30)
# 4. Crop: width = frame_height * (9/16), centered on smoothed_x
# 5. Write to cv2.VideoWriter at same fps

# Merge audio back:
ffmpeg -i reframed_video.mp4 -i original.mp4 -map 0:v -map 1:a -c:a copy final.mp4

General Fallback — Smart Center Crop

When no subject detected, apply center crop biased slightly upward. Subjects are rarely at the very bottom of frame. Safe for most content types.
